Canonical reference

The Zuzo design system.

Every new feature, page, or component must be built from these tokens and patterns. The rule of thumb: a white canvas, near-black text, and orange used sparingly as a single point of emphasis. When in doubt, copy a block from this page.

01 · Color

A neutral canvas with one warm accent.

Roughly 90% of any screen is white, near-black, and grey. Orange is reserved for one or two focal moments per section: primary actions, key figures, live signals, and hover states. Never fill large areas with orange.

Use: orange for primary buttons, the live dot, key metric values, and hover borders. Avoid: orange backgrounds larger than an icon well, orange body text, and more than one orange element competing in the same eyeline.

02 · Typography

Inter for everything, JetBrains Mono for data.

Headings are light weight (300) at large sizes with tight negative tracking, which reads premium on white. Body is regular weight in grey. Monospace carries labels, metrics, addresses, and any machine output. Sentence case throughout — no all-caps headlines.

03 · Buttons

Fully-rounded pills, three weights.

Primary (orange) for the single most important action in a view. Dark for app entry and high-intent navigation. Grey for everything secondary. One primary button per section, never two side by side.

.btn.btn-primary · .btn.btn-dark · .btn.btn-grey — radius --r-pill, hover lifts 1px with a soft shadow.

04 · Header & navigation

One header. Two variants. Never inconsistent.

Every page uses the same sticky nav: brand on the left, the same five section links in the centre, and a single dark "Launch app" pill on the right. Marketing pages add a thin dark announcement strip above the nav; the order-book app replaces that strip with its live status bar. The nav links, CTA style, and order never change between pages.

05 · Radius & layout tokens

Consistent rounding and rhythm.

Container maxes at --max (1320px) with responsive gutters via --gut. Use the radius scale below rather than ad-hoc pixel values.

06 · Voice & usage rules

How to keep new work on-system.

Do

  • Write full, concise sentences in institutional voice.
  • Lead with the figure or claim, then explain it.
  • Keep one orange focal point per section.
  • Use monospace for any data, metric, or address.
  • Reuse these classes; extend tokens, don't replace them.

Don't

  • Use choppy two-word fragments as copy.
  • Add new accent colors or gradients outside the tokens.
  • Fill large areas with orange or use pure black text.
  • Stack two primary buttons in one view.
  • Introduce a third typeface or all-caps headlines.

Every screen should pass three checks before shipping: the canvas is predominantly neutral, the copy reads as complete institutional sentences, and the components are reused from this page rather than re-styled. Tokens live in the :root block at the top of each HTML file.
